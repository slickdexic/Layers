Drop Shadow Fix Validation Report

‚úÖ Issue Resolution Summary

The "Save failed: Error saving layers" issue when using drop shadow checkbox has been fixed by updating both the backend API validation and the frontend client-side validation.

üîß What Was Fixed

1. Backend API Validation (ApiLayersSave.php)

Added missing properties to $allowedProps array:
  • 'shadow' => 'boolean' - Drop shadow toggle
  • 'shadowBlur' => 'numeric' - Shadow blur amount
  • 'shadowOffsetX' => 'numeric' - Shadow X offset
  • 'shadowOffsetY' => 'numeric' - Shadow Y offset
  • 'shadowSpread' => 'numeric' - Shadow spread/size
  • 'blurRadius' => 'numeric' - For blur layers
  • 'radiusX' => 'numeric', 'radiusY' => 'numeric' - For ellipse layers
  • 'arrowSize' => 'numeric', 'arrowStyle' => 'string' - For arrow layers
  • 'x1' => 'numeric', 'y1' => 'numeric', 'x2' => 'numeric', 'y2' => 'numeric' - Line coordinates

2. Added 'blur' Layer Type

Added to $validTypes array: 
'text', 'arrow', 'rectangle', 'circle', 'ellipse', 'polygon', 'star', 'line', 'highlight', 'path', 'blur'

3. Frontend Client-side Validation (LayersValidator.js)

Added validation for shadow properties:
  • shadowBlur - Range: 0-100
  • shadowOffsetX - Range: -100 to 100
  • shadowOffsetY - Range: -100 to 100
  • shadowSpread - Range: 0-50
  • arrowSize - Range: 1-100

4. Improved Boolean Handling

Enhanced PHP validation to accept various boolean representations from JavaScript:

// Accept: true, false, 1, 0, '1', '0', 'true', 'false'
if ( $type === 'boolean' ) {
    if ( is_bool( $value ) || $value === 1 || $value === 0 || 
         $value === '1' || $value === '0' || $value === 'true' || $value === 'false' ) {
        $isValid = true;
        $value = (bool)$value; // Normalize to proper boolean
    }
}

üîç Root Cause Analysis

The issue was a validation mismatch between frontend and backend:

Frontend (LayerPanel.js) was setting:

self.editor.updateLayer( layer.id, { 
    shadow: true,
    shadowColor: '#000000',
    shadowBlur: 5,
    shadowOffsetX: 2,
    shadowOffsetY: 2,
    shadowSpread: 1
});

But validation layers were missing:

  • Client-side: LayersValidator.js had no validation rules for shadow properties
  • Server-side: ApiLayersSave.php $allowedProps array was missing shadow properties

Result:

Frontend validation failed first (blocking the API call), and even if it passed, server validation would have failed, causing the "Save failed" error.
